HTML Tables

Tables in HTML allow you to display data in rows and columns, making it easy to organize and present information. Tables are constructed using the <table> element, with rows created using <tr> and cells with <td> for data cells and <th> for table headers.

Basic Table

Basic Table Example
Header 1 Header 2 Header 3
Data 1 Data 2 Data 3
Data 4 Data 5 Data 6

Table Structure

A table structure consists of:

  • <table>: The container for the entire table.
  • <tr>: Represents a table row.
  • <th>: Represents a table header cell.
  • <td>: Represents a table data cell.

Table Styling

You can apply CSS styles to tables for better presentation. For example:

table {
    width: 100%;
    border-collapse: collapse;
}

th, td {
    border: 1px solid #dddddd;
    padding: 8px;
    text-align: left;
}

th {
    background-color: #f2f2f2;
}

Advanced Table

Advanced Table Example
Name Age Email
John Doe 25 john@example.com
Jane Smith 30 jane@example.com

With tables, you can display tabular data like product lists, schedules, and much more. Remember to use tables only for data presentation and avoid using them for layout purposes, as it may affect accessibility and responsiveness.

> Images and Forms


HTML Image Tags

Images can make your web pages visually appealing and engaging. In HTML, you can display images using the <img> tag. The <img> tag is an empty element, meaning it does not have a closing tag.


Basic Image

To display an image, use the <img> tag with the src attribute, which specifies the image file's URL or path:

<img src="path/to/image.jpg" alt="Image description>

The alt attribute provides alternative text for screen readers and is important for accessibility purposes.


Image Attributes

The <img> tag supports several attributes:

  • src: Specifies the image URL or path.
  • alt: Provides alternative text for screen readers and search engines.
  • width and height: Sets the image dimensions in pixels.
  • title: Adds a tooltip when hovering over the image.
  • align: Specifies the image alignment (left, right, center).
  • border: Sets the border thickness around the image.

Example

<img class="imgexplainimg" src="path/to/image.jpg" alt="Example Image" width="300" height="200" title="This is an example image" border="1" align="left" />

Image Formats

Common image formats supported by HTML include JPEG (.jpg), PNG (.png), GIF (.gif), and SVG (.svg). Choose the appropriate format based on your image's characteristics and requirements.

Lazy Loading

Consider using the "loading" attribute to enable lazy loading for images that are not initially visible on the page. This can improve page loading speed and performance.

<img src="path/to/image.jpg" alt="Lazy Loaded Image" loading="lazy">

Image Maps

HTML also supports image maps, allowing you to define clickable areas on an image linked to different URLs or actions:

<img src="path/to/image.jpg" alt="Image Map" usemap="#mapname">

HTML Form tag


Forms are essential for gathering user input on web pages. In HTML, you create forms using the <form> element, and various input elements collect user data. Forms allow users to submit data to servers for processing or interaction with web applications.


Basic Form Structure :

To create a basic form, use the <form> tag with the action attribute, which specifies the server-side script or URL to handle form data upon submission:

        <form action="submit_form.php" method="post">
            <!-- Form inputs will go here -->
        </form>

The method attribute defines the HTTP method for form submission, typically get or post. Data submitted via post is not visible in the URL, making it more secure for sensitive information.


Form Input Elements

Various input elements are available to collect different types of user data:


    Inputs
  • <input type="text">: Single-line text input.
  • <input type="password">: Password input (masked characters).
  • <input type="email">: Email address input with built-in validation.
  • <input type="number">: Numeric input with optional min-max values.
  • <input type="checkbox">: Checkbox for multiple-choice selection.
  • <input type="radio">: Radio buttons for single-choice selection.
  • <input type="submit">: Submit button to send form data.
  • <input type="reset">: Reset button to clear form data.
  • <textarea>: Multi-line text input for longer text.

    • Actions
  • <button>: Customizable button for form interactions.
  • <select>: Drop-down menu for single or multiple selections.

> HTML vs HTML5


HTML HTML5
Stands for HyperText Markup Language. The latest version of HTML.
Introduced in the early 1990s. Released in 2014.
Had limited elements and lacked modern features. Introduced new semantic elements like header, footer, nav, etc.
Didn't support multimedia without plugins (e.g., Flash). Supports native multimedia elements like <video> and <audio>.
Styling was done mainly with CSS. Has new styling options, like <style> within <body>.
Deprecated elements like <center> and <font>. Deprecated elements removed, replaced by CSS and semantic tags.
No canvas element for graphics. Introduced <canvas> for interactive graphics and animations.
Less emphasis on form validation. Introduced form validation attributes like 'required' and 'pattern'.
No support for responsive design. Designed with responsiveness in mind.
